# 시흥링크
시흥시 바이오 산업단지 커뮤니티 플랫폼 - 백엔드 시스템

시흥링크는 시흥시 바이오 산업단지 내 구성원들을 대상으로 정보 공유, 커뮤니티 소통, 사용자 인증 기반 서비스 제공을 목표로 한 웹 플랫폼입니다.

본 프로젝트에서는 저는 백엔드 개발자로 참여해,
사용자 인증/인가 흐름과 핵심 도메인 API 설계 및 구현을 담당했습니다.

## 1. 프로젝트 배경 및 요구사항(가상)

프로젝트의 주요 요구사항은 다음과 같습니다.

- 사용자 인증을 기반으로 한 커뮤니티 서비스 제공
- 로그인 여부 및 권한에 따라 접근 가능한 기능 구분
- 프론트엔드와 분리된 REST API 구조
- 팀 프로젝트 환경에서 역할 분담이 가능한 백엔드 구조

  특히, 단순 CRUD 제공을 넘어 인증 상태에 따라 기능이 제어되는 구조를 안정적으로 구현하는 것이 중요했습니다.

## 2. 전체 시스템 구조

  - Frontend: React
  - Backend: Spring Boot 기반 REST API
  - Database: MySQL
  - 인증 방식: JWT 기반 인증
 
    프론트엔드와 백엔드는 명확히 분리된 구조로 설계되었으며,
    백엔드는 인증/인가 및 비즈니스 로직에 집중하도록 구성했습니다.

## 3. 핵심 기능 및 동작 흐름

  ### 사용자 인증 흐름

  1. 사용자가 로그인 요청을 보냅니다.
  2. 인증 성공시 JWT Access Token을 발급합니다.
  3. 이후 모든 보호된 API 요청은 Authorization Header를 통해 토큰을 전달받아 처리합니다.
  4. 인증되지 않은 사용자는 인증 요청을 해야지만 서비스에 접근할 수 있도록 설정했습니다.
  5. 인증이 필요한 요청에 토큰이 없거나 유효하지 않을 경우 요청을 거절합니다.
 
  이 흐름을 통해 로그인 상태에 따른 접근 제어를 명확히 분리했습니다.

  ## 4. 설계 시 고려했던 핵심 포인트

  ### 인증을 중심으로 한 API 보호

  커뮤니티 서비스 특성상,
  - 글 작성, 수정, 삭제
  - 사용자 관련 정보 접근과 같은 기능은 반드시 인증된 사용자만 접근 가능해야 했습니다.
 
  이를 위해 인증 로직을 각 컨트롤러에 흩어놓지않고, Spring Security 필터 단에서 인증을 일관되게 처리하도록 구성했습니다.

  ## 5. 주요 API 예시
  ### 로그인
  POST /v1/auth/login
  
  ### 게시글 작성
  POST /v2/articles/create
  Authorization: Bearer {accessToken}

  동작 규칙
  * 인증된 사용자만 접근 가능
  * 토큰이 없거나 유효하지 않으면 요청 거절

  ### 게시글 조회
  GET /articles

## 6. 협업 환경에서의 역할과 기여
  본 프로젝트는 프론트엔드와 백엔드가 분리된 팀 프로젝트였으며,
  저는 다음 영역을 중심으로 기여했습니다.
  * 백엔드 API 구조 설계
  * JWT 기반 인증/인가 로직 구현
  * 사용자 및 커뮤니티 도메인 API 개발
  * 프론트엔드 연동을 고려한 요청/응답 규격 정의
  * 인증 실패 및 예외 상황 처리

## 7. 기술 스택
  * Java
  * Spring Boot
  * Spring Security
  * JWT
  * JPA
  * MySQL

## 8. 프로젝트를 통해 배운 점
  이 프로젝트를 통해,
  단순히 기능을 구현하는 것을 넘어
  팀 환경에서 백엔드가 어떤 책임을 가져야 하는지를 체감할 수 있었습니다.

  특히 인증과 권한 제어는
  개별 기능이 아니라 서비스 전반에 영향을 미치는 요소이기 때문에,
  초기에 구조를 명확히 잡는 것이 중요하다는 점을 배웠습니다.

  이 경험을 이후 개인 프로젝트에서도
  설계 단계에서 인증 흐름과 접근 제어를 먼저 정의하는 기준으로 삼고 있습니다.
